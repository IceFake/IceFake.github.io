P2602 [ZJOI2010] 数字计数\color{purple}{P2602\ [ZJOI2010]\ 数字计数}P2602 [ZJOI2010] 数字计数 record
dp[pos][cnt][num] 进行记忆化，每次dfs(pos, num, cnt, pre, limit)，其中 pos 表示当前位数，num 表示需要搜索的数字，cnt 表示当前搜索到了多少个该数字，pre 表示有无前导零，limit 表示是否到达上界。
至于转移——

Rep(i,0,up) tem += dfs(pos - 1, num, cnt + ((i == num) && (!pre || i)), !i && pre, (i == up) && limit);

P4127 [AHOI2009] 同类分布\color{purple}{P4127\ [AHOI2009]\ 同类分布}P4127 [AHOI2009] 同类分布 record
dp[pos][sum][sta] 进行记忆化，每次dfs(pos, sum, sta, mod, limit)，其中 pos 表示位数，sum 表示数字和， sta 表示原数模 mod 之后的大小，limit 表示是否到达上界。
于是我们只需要

Rep(i,0,up) tem += dfs(pos - 1, sum + i, (sta * 10 + i) % mod, mod, (i == up) && limit);

进行记忆化搜索就ok
而当 pos == 0 且 sum 与 mod 相等时 返回 1。

P4317 花神的数论题\color{purple}{P4317\ 花神的数论题}P4317 花神的数论题 record
稍微需要一点点转化，我们考虑每次求出范围内含有 tot (1≤tot≤pos1 \leq tot \leq pos1≤tot≤pos) 个 1 的二进制数有多少个，再用快速幂得出乘积。
于是，我们可以用dp[pos][sum][tot]进行记忆化，每次dfs(pos, sum, limit, tot)，其中 pos 表示当前位数，sum 表示到当前位有多少个 1 ，limit 表示是否到达上界。
转移就可以这样写：

Rep(i,0,up) tem += dfs(pos - 1, sum + i, (i == up) && limit, tot);
